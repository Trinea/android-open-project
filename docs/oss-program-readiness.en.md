# Claude for OSS & Codex for OSS Readiness

## Project Overview
- **Project**: Android Open Project catalog (`android-open-project`)
- **Mission**: curate up-to-date Android OSS references. The GitHub repo, timeshining.com, and the Dev Tools App list 30k+ libraries with categorization, search, and recommendation features.
- **Audience**: Android developers, technical communities, and training/education partners.

## Governance & Compliance
- **License**: Apache License 2.0 (compatible with Claude/Codex requirements).
- **Maintainer**: [Trinea](https://github.com/Trinea) — oversees crawlers, website/app content, documentation, and community ops.
- **Contribution paths**: GitHub issues/PRs, the submission form on timeshining.com, and QQ/WeChat groups. Every change is reviewed before landing.
- **Community channels**: CodeKK WeChat account, QQ group, Weibo @Trinea, Dev Tools App notifications — allow rapid security/compliance responses.

## Impact & User Value
- 10+ years of continuous operation, 38k+ GitHub stars, and 150k+ monthly active developers on the website/app.
- Referenced by tech communities, training courses, and corporate knowledge bases to assess OSS maturity quickly.
- Automated push and curated topics increase visibility for local Android OSS projects.

## Claude for OSS Plan
1. **Knowledge expansion**: use Claude to summarize new projects, highlight risks, and auto-generate category blurbs to shorten curation time.
2. **Community support**: Claude assists in answering FAQs in issues/discussions with bilingual responses.
3. **Governance**: leverage Claude to check licenses and security statements within submissions to keep data trustworthy.

## Codex for OSS Plan
1. **Automation scripts**: Codex helps build scripts that sync GitHub metadata with timeshining.com and the Dev Tools App.
2. **Quality checks**: auto-generate crawler tests and link-health checks to reduce manual regression effort.
3. **Modern infrastructure**: remodel the data schema and expose JSON/GraphQL outputs for downstream systems with Codex-generated tooling.

## Responsible AI Use
- Human maintainers review every AI-assisted change and keep audit logs.
- No private, unpublished, or copyright-restricted data is uploaded to AI tools.
- All AI-generated content is labeled, and contributors must cite original sources.
- Feedback channels (issues, QQ, WeChat) allow developers to report mistakes for fast fixes.

## Support Needs
1. Automate knowledge-base building, classification, and translation workflows.
2. Produce daily/weekly OSS intelligence briefings for hundreds of thousands of developers.
3. Enhance search/recommendation algorithms and public APIs on timeshining.com.
